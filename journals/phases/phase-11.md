### Задачи фазы

#### Оркестратор (ты)
Передаёшь `frontend-web-client` задачу на реализацию караоке-плеера — самого сложного UI-компонента. Ключевая фича — послоговая подсветка текста в реальном времени, синхронизированная с аудио. Плеер использует `<audio>` элемент для инструментала + текстовый overlay для субтитров (не `<video>`, рендер субтитров на клиенте для максимального контроля). После реализации — `ui-design-director` для ревью.

#### Подагент `frontend-web-client`
Реализует караоке-плеер по спеке `06_karaoke_player.md`:

1. **PlayerPage** (`src/pages/PlayerPage/`):
   - Полноэкранный тёмный режим
   - При входе: `POST /queue/{entry_id}/start` → получает `clip_url`, `syllable_timings`, `duration_sec`

2. **KaraokePlayer** (`src/components/KaraokePlayer/`):
   - `<audio>` элемент для воспроизведения инструментальной дорожки (HTTP Range Request)
   - Текстовый overlay поверх тёмного фона (не video-тэг для субтитров)

3. **LyricHighlight** (`src/components/LyricHighlight/`):
   - Главный компонент: послоговая подсветка в реальном времени
   - Получает `syllable_timings: [{syllable, start, end}]` из API
   - Каждый кадр (requestAnimationFrame):
     - Читает `audio.currentTime`
     - Находит текущий слог: `syllables.find(s => s.start <= currentTime < s.end)`
     - 3 состояния слогов (из `06_karaoke_player.md`):
       - Спетые (прошедшие): dim, `rgba(255,255,255,0.25)`
       - Активный (текущий): neon glow `#F0ABFC`, text-shadow `0 0 32px`, font-size 72px
       - Предстоящие: белый `rgba(255,255,255,0.95)`
   - Строки текста: показ 2-3 строк одновременно, плавный скролл при переходе
   - Шрифт: Inter 700, 72px для активной строки

4. **Управление воспроизведением**:
   - Play/Pause (большая кнопка по центру)
   - Перемотка ±15 сек (кнопки-стрелки)
   - Громкость (слайдер)
   - Таймлайн-слайдер (прогресс-бар с возможностью перемотки)
   - При перемотке — подсветка мгновенно синхронизируется с новой позицией

5. **Завершение**:
   - Кнопка «Завершить» (верхний правый угол)
   - По окончании аудио (event 'ended') или по кнопке → `POST /queue/{entry_id}/finish`
   - Переход обратно к QueuePage с информацией о следующем участнике

#### Подагент `ui-design-director`
Ревью: открывает плеер, проверяет послоговую подсветку, цвета (dim/neon/white), размер шрифта, плавность анимации, управление воспроизведением.

#### Пользователь
Тестирует караоке на реальном треке: подсветка, перемотка, пауза, громкость, завершение. Подтверждает или вносит замечания.

### Выходные артефакты
- PlayerPage с полноэкранным караоке-плеером
- LyricHighlight: послоговая подсветка в реальном времени (dim / neon glow / white)

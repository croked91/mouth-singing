## Фаза 10a: Фронтенд — QueuePage + рекомендации

### Входные артефакты
- Результат Фазы 9 (React-приложение с Landing + Sessions)
- `design/prompts/03_main_queue.md` — экран очереди с текущим исполнителем, списком очереди, табами
- `journals/ARCHITECTURE.md` — раздел 10 «API-контракт» (Queue, Recommendations)

### Задачи фазы

#### Оркестратор (ты)
Передаёшь `frontend-web-client` задачу на реализацию главной страницы очереди — центрального экрана приложения. Здесь отображается текущий исполнитель, очередь участников, и табы для поиска/рекомендаций/загрузки. В этой фазе реализуется QueuePage и таб «Рекомендации». Табы «Поиск» и «Загрузить» — в Фазе 10b.

#### Подагент `frontend-web-client`
Реализует QueuePage по спеке `03_main_queue.md`:

1. **QueuePage** (`src/pages/QueuePage/`):
   - Левая колонка (или верхняя секция): текущий исполнитель + очередь
     - «Сейчас поёт» — карточка текущего участника + трек
     - Кнопка «Петь!» → переход к PlayerPage
     - Список очереди: позиция, участник, трек, кнопка «Пропустить»
   - Правая колонка (или табы): Поиск / Рекомендации / Загрузить трек
   - Обновление через polling `GET /sessions/{id}/queue` каждые 5 сек

2. **Таб «Для вас» (Рекомендации)**:
   - Вызывает `GET /recommendations?participant_id=X&session_id=Y`
   - Отображает карточки треков (TrackCard): обложка-плейсхолдер, артист, название, duration
   - Кнопка «Выбрать» на карточке → `POST /queue` → трек добавляется в очередь
   - Показывает стратегию: «Популярное», «Похожее на ...», «На основе вашей сессии»

3. **Компоненты**:
   - `TrackCard` — карточка трека (glassmorphism card, hover-эффект)
   - `QueueItem` — элемент очереди (позиция, аватар, имя, трек)
   - `ParticipantChip` — чип участника (цветной, маленький)
   - Все компоненты стилизованы по `00_design_system.md`

4. **Табы «Поиск» и «Загрузить»** — пустые заглушки с надписью «Скоро» (будут в 10b)

#### Подагент `ui-design-director`
Ревью: проверяет QueuePage на соответствие `03_main_queue.md`, корректность glassmorphism, отступов, типографики.

#### Пользователь
Проверяет очередь, рекомендации, выбор треков. Подтверждает или вносит замечания.

### Выходные артефакты
- QueuePage с текущим исполнителем и очередью
- Таб «Рекомендации» с карточками треков
- Компоненты: TrackCard, QueueItem, ParticipantChip
- Пропуск очереди через UI
- Выбор трека → добавление в очередь
- Коммит

## Фаза 12: Фронтенд — Админка и UX polish

### Входные артефакты
- Результат Фаз 9-11 (полное React-приложение)
- `design/prompts/07_admin_modal.md` — подробная спецификация админ-модала: PIN-ввод (numpad), 4 состояния (PIN entry, wrong PIN, unlocked, confirmation)
- `design/prompts/00_design_system.md` — для финального polish

### Задачи фазы

#### Оркестратор (ты)
Передаёшь `frontend-web-client` задачу на реализацию админ-панели и финальную шлифовку UX. Админ-модал — overlay поверх любого экрана с PIN-кодом и кнопкой завершения сессии. Также нужно обработать все edge cases и ошибки. После — `ui-design-director` для финального ревью.

#### Подагент `frontend-web-client`
Реализует:

1. **AdminModal** (`src/components/AdminModal/`) — по спеке `07_admin_modal.md`:
   - Overlay поверх текущего экрана: backdrop blur(24px), dimming `rgba(5,5,8,0.7)`
   - Модальная карточка: 460px, glassmorphism, border-radius 28px
   - **State A: PIN Entry** — 4 PIN-dot индикатора + виртуальный numpad (3x4 grid):
     - Цифры 0-9, backspace, confirm
     - Dot states: empty, filled (gradient), active (pulse)
   - **State B: Wrong PIN** — красные dots, shake-анимация, «Неверный PIN»
   - **State C: Unlocked** — PIN dots green, кнопка «ЗАВЕРШИТЬ СЕССИЮ» (красная, destructive)
   - **State D: Confirmation** — «Завершить сессию?» + «Очередь будет очищена...» + «Отмена» / «Да, завершить»
   - При подтверждении: `DELETE /sessions/{id}` с X-Admin-Secret → возврат на WelcomePage

2. **Обработка ошибок** по всему приложению:
   - Сетевые ошибки: «Нет подключения к серверу, попробуйте позже»
   - 4xx: осмысленные сообщения на русском
   - 5xx: «Что-то пошло не так, попробуйте позже»
   - Таймауты: «Сервер не отвечает»
   - Toast-нотификации (MUI Snackbar) для ошибок и успешных действий

3. **Edge cases**:
   - Пустая очередь: «Добавьте трек через поиск или загрузку»
   - Нет результатов поиска: «Ничего не найдено»
   - Таймаут обработки трека (> 120 сек): предупреждение + кнопка «Отменить»
   - Потеря SSE-соединения: автоматическое переподключение

4. **Адаптивность**:
   - 1920x1080 (основное разрешение — монитор/ТВ в кабинке)
   - 1280x800 (планшет)

5. **Skeleton loading states**: shimmer-эффект при загрузке данных (карточки треков, очередь, рекомендации)

#### Подагент `ui-design-director`
Финальное ревью всего приложения: все экраны, состояния, анимации, edge cases, адаптивность.

#### Пользователь
Тестирует admin flow (PIN → завершение), проверяет обработку ошибок, адаптивность. Подтверждает или вносит замечания.

### Выходные артефакты
- AdminModal с 4 состояниями (PIN, error, unlocked, confirmation)
- Обработка всех ошибок с русскоязычными сообщениями
- Edge cases обработаны
- Адаптивность 1920x1080 + 1280x800
- Skeleton loading states
- Нет белых экранов или необработанных исключений

Схема обработки загруженного MP3-файла воркером в режиме `make up-gpu`.
Замеры на реальном треке (~3:18, русский язык, 2026-03-31).

## Управление VRAM

Модели загружаются и выгружаются последовательно (lazy load + cleanup):

```
UVR load (~1с) → separate → UVR cleanup (~0.13с) → VRAM свободна
Whisper (предзагружен) → transcribe → Whisper cleanup (~0.18с) → VRAM свободна
CTC load (~2.6с) → align (0.75с) → CTC cleanup (~0.17с) → VRAM свободна
Embedder (резидентно, ~850 МБ)
```

Пиковое потребление VRAM на воркер: ~4.3 ГБ (CTC MMS-300M fp16).
При 2 воркерах + десктоп: ~10.2 ГБ из 16 ГБ.

## Наблюдения

- **Главное бутылочное горлышко** — Lyrics Search (38.52с, 39%). CTC больше не узкое место после переноса на GPU.
- **CTC ускорение: 36x** — с 27с (ONNX CPU subprocess) до 0.75с (PyTorch GPU fp16 in-process).
- **Параллельная секция** (шаги 2-4): wall-clock ~28.53с. Ветка B (VAD 1.11с + Whisper 27.42с) длиннее ветки A (24.56с).
- **Load/cleanup overhead** всех моделей — 4.2с (4.3% пайплайна). Оправдано vs риск OOM при резидентном хранении.
- **Общее ускорение: -44%** (175.66с → 98.73с) по сравнению с baseline.

# Мастер промпт караоке

## Описание проекта

Караоке веб-приложение для кабинок в караоке-клубах. Группа гостей запускает сессию, ищет треки, получает рекомендации по настроению и поёт по очереди с послоговой подсветкой текста. Поддерживается загрузка пользовательских MP3 с автоматической генерацией караоке.

## Текущее состояние

Приложение реализовано и задеплоено. Каталог ~17 000 треков. Все основные функции работают:
- Сессии, очередь, поиск (FTS5 + семантический)
- Кластерные рекомендации с тегами настроения, popularity scoring, MMR diversity
- Загрузка пользовательских треков (GPU и API режимы worker)
- Караоке-плеер с послоговой подсветкой (CTC Hybrid alignment)
- Админка (PIN-авторизация)

## Технологический стек

- **Backend:** Python + FastAPI + aiosqlite + QDrant v1.16.2
- **Frontend:** React 19 + TypeScript + MUI 7 + Zustand
- **Worker (GPU):** BS-Roformer + faster-whisper + CTC Forced Aligner + OpenAI gpt-4o-mini
- **Worker (API):** MVSEP + OpenAI Whisper + CTC Forced Aligner + OpenAI gpt-4o-mini
- **Деплой:** Docker Compose (GPU и API варианты), Makefile

## Требования (user stories)

### Гость караоке-клуба
- Запуск сессии без регистрации, автоматическая генерация никнеймов
- Поиск треков по словам из песни
- Рекомендации по настроению через теги, без регистрации
- Загрузка собственного MP3 и получение караоке
- Управление воспроизведением (перемотка, пауза, громкость)
- Послоговая подсветка текста
- Тёмная тема интерфейса

### Администратор
- Быстрое завершение сессии через PIN-авторизацию
- Интуитивно понятный интерфейс

### Владелец сервиса
- Массовый бутстрап каталога (17 000+ треков)
- Минимальные затраты на содержание
- Деплой в одну команду (`make up-gpu` / `make up-api`)
- Рекомендации без пользовательских ID и регистраций

## Принципы работы

- Код находится в корне репозитория: `backend/`, `worker/`, `frontend/`, `shared/`, `scripts/`

- При потере контекста — начинай с `journals/PHASES.md` и `journals/PROJECT_LOG.md`

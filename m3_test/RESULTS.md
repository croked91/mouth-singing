# M3 Test — Результаты экспериментов по слоговой разметке

**Дата**: 2026-03-06

## Задача
Найти лучший способ получения послоговых таймингов для караоке при наличии правильного текста песни, но без LRC-таймингов.

## Тестовые треки
| # | Артист | Название | Язык |
|---|--------|----------|------|
| 1 | Слава КПСС | Владимир Путин | ru |
| 2 | Григорий Лепс | Орлы или вороны | ru |
| 3 | Red Hot Chili Peppers | Bullet Proof | en |
| 4 | Дима Билан | Я просто люблю тебя | ru |
| 5 | Король и Шут | Помнят с горечью древляне | ru |

Эталон — syllable_timings из БД (путь 1 бутстрапа: LRC + WhisperX force_align).

## Протестированные методы

### 1. Baseline (текущий fallback в бутстрапе)
WhisperX ASR → pyphen пропорциональное деление слов на слоги.

### 2. V2: WhisperX ASR → fuzzy match → force_align
WhisperX ASR для пословных таймингов → difflib сопоставление с известным текстом → построчные сегменты → WhisperX force_align слогов.

### 3. V3b: WhisperX ASR → difflib alignment → pyphen
Как V2, но без force_align — пропорциональное деление внутри слов.

### 4. V3: Sonoix + GPT-4o-mini LLM-коррекция
Sonoix API → BPE-токены → LLM корректирует текст токенов.

### 5. CTC Word-level
CTC Forced Aligner (MMS-300m ONNX) с известным текстом → пословные тайминги → pyphen.

### 6. CTC Char-level (full track)
CTC Forced Aligner с split_size="char" на весь трек.

### 7. CTC Hybrid (ЛУЧШИЙ)
CTC word-level на весь трек → char-level CTC внутри каждого слова → слоги из char-таймингов.

## Провалы
- **V3 Sonoix + LLM**: MAE 13-54с. GPT-4o-mini не умеет переписывать BPE-токены с сохранением таймингов
- **CTC Char (full track)**: MAE 2-24с. Накапливающийся дрейф на 1700+ символах
- **Трек 3 (RHCP)**: провал у всех методов — в бутстрапе обработан как language="ru"

## Выводы

1. **CTC Hybrid — лучший подход**: лучший MAE (0.240с), hit rate на уровне WhisperX (71% vs 72%)
2. **Не требует ASR** — текст подаётся напрямую, нет зависимости от качества распознавания
3. **Один проход** лёгкой ONNX-модели (MMS-300m), работает на CPU за ~22с/трек
4. **LLM-коррекция BPE-токенов не работает** — задача слишком сложна для языковой модели
5. **Пропорциональное деление** (pyphen) — главный bottleneck всех word-level методов
